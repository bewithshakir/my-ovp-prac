    breakpoint

    @mixin breakpoint($point) { ... }
    View source

    Description

    Outputs the appropriate media query (breakpoint) based on the passed $point parameter.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $point

    Supported values: webkit, sm, md, lg, xl, ipad-portrait, ipad-landscape

    		String—none

    Content

    This mixin allows extra content to be passed (through the @content directive).

    Example

    Basic Usage Sass

    .widget {
    color: red;

    @include breakpoint("sm") {
        color: blue;
    }
}

    Basic Usage CSS Output

    .widget {
    color: red;
}

@media (min-width: 768px) {
    .widget {
        color: blue;
    }
}

    Throws

    • @warn if $point is not a supported value.

    Used by

    • [mixin] viewport-units

    • Christopher Dura @dura

    twcicon-svg

    @mixin twcicon-svg($icon, $state: null) { ... }
    View source

    Description

    Outputs an icon class selector with a URL to the appropriate icon image. Generally used to change colors (state) of icons when hovering or focusing a parent element, etc.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $icon

    The root icon name of SVG, e.g. 'lock' will resolve to ../path/to/icons/lock.svg

    		String—none
    $state

    A semantic state (color) descriptor, e.g. 'danger' will resolve to ../path/to/icons/lock-danger.svg

    		Stringnull

    Example

    Basic Usage Sass

    .widget {
    @include twcicon-svg('key');
    color: red;

    &:hover {
        @include twcicon-svg('key', 'danger');
    }
}

    Basic Usage CSS Output

    .widget {
    color: red;
}

.widget .twcicon-svg-key {
    background-image: url("../images/icons/twcicon-key.svg");
}

.widget:hover .twcicon-svg-key {
    background-image: url("../images/icons/twcicon-key-danger.svg");
}

    viewport-units

    @mixin viewport-units($property, $value) { ... }
    View source

    Description

    VH and VW units can cause issues on iOS devices. To overcome this, create media queries that target the width, height, and orientation of iOS devices. It isn't optimal, but there is really no other way to solve the problem.

    Parameters

    parameterNameparameterDescriptionparameterTypeparameterDefault value
    $property

    An unquoted CSS property.

    		String—none
    $value

    Value in vw or vh units.

    		Number—none

    Example

    Basic Usage Sass

    .widget {
    @include viewport-units(min-height, 75vh);
}

    Basic Usage CSS Output

    .widget {
    min-height: 75vh;
}

@media (min-device-width: 768px) and (max-device-width: 1024px) and (orientation: portrait) {
    .widget {
        min-height: 768px;
    }
}

@media (min-device-width: 768px) and (max-device-width: 1024px) and (orientation: landscape) {
    .widget {
        min-height: 576px;
    }
}

    Requires

    • [mixin] breakpoint
    • [function] strip-unit

    • iOS Resolution Quick Reference
    • http://stephen.io/mediaqueries/
